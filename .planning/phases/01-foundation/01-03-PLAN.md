Create tests/harness/run-isolated.sh (chmod +x).

Purpose: Test a single pipeline component in isolation. In Phase 1, every component is a passthrough to the reference binary. In later phases, the component-specific path is replaced with the generated .fun equivalent.

Interface:
  run-isolated.sh --lexer <input-file>
  run-isolated.sh --indent-filter <input-file>
  run-isolated.sh --parser <input-file>
  run-isolated.sh --lexer --expr "<expression>"
  (same --expr flag for the other modes)

Behavior in Phase 1 (passthrough):
- All three modes call the reference binary with --emit-tokens or --emit-ast
- The output is printed to stdout
- The script documents (in comments) where the generated component would be wired in

Full script:
```bash
#!/usr/bin/env bash
# run-isolated.sh — per-component isolation runner
#
# Usage:
#   run-isolated.sh --lexer    <input-file>
#   run-isolated.sh --indent-filter <input-file>
#   run-isolated.sh --parser   <input-file>
#   Add --expr "<expression>" instead of <input-file> for expression mode.
#
# In Phase 1 all components are passthroughs to the reference binary.
# In later phases replace the LEXER_BIN / INDENT_FILTER_CMD / PARSER_BIN
# variables with the generated component paths.
#
# The LANGTHREE_BIN env var overrides the reference binary.

set -euo pipefail

LANGTHREE_BIN="${LANGTHREE_BIN:-/Users/ohama/vibe-coding/LangThree/src/LangThree/bin/Release/net10.0/LangThree}"
SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
PROJECT_ROOT="$(cd "$SCRIPT_DIR/../.." && pwd)"

# --- Component overrides (replace in later phases) ---
# Phase 2: set INDENT_FILTER_CMD to run IndentFilter.fun
# Phase 3: set LEXER_BIN to run the generated Lexer.fun
# Phase 6: set PARSER_BIN to run the generated Parser.fun
LEXER_BIN="${LEXER_BIN:-$LANGTHREE_BIN}"
PARSER_BIN="${PARSER_BIN:-$LANGTHREE_BIN}"

MODE="${1:?Usage: run-isolated.sh --lexer|--indent-filter|--parser [--expr expr | input-file]}"
shift

case "$MODE" in
    --lexer)
        # Phase 1: passthrough — real Lexer.fs via reference binary
        # Phase 3+: replace with: langthree "$PROJECT_ROOT/src/Main.fun" --lexer "$@"
        "$LEXER_BIN" --emit-tokens "$@"
        ;;
    --indent-filter)
        # Phase 1: passthrough — IndentFilter.fs via reference binary
        # Phase 2+: replace with: langthree "$PROJECT_ROOT/src/IndentFilter/IndentFilter.fun" "$@"
        "$LANGTHREE_BIN" --emit-tokens "$@"
        ;;
    --parser)
        # Phase 1: passthrough — Parser.fs via reference binary
        # Phase 6+: replace with: langthree "$PROJECT_ROOT/src/Main.fun" --parser "$@"
        "$PARSER_BIN" --emit-ast "$@"
        ;;
    *)
        echo "ERROR: Unknown mode '$MODE'. Use --lexer, --indent-filter, or --parser."
        exit 1
        ;;
esac
```

Note: In Phase 1, --indent-filter simply calls the reference binary (which runs the full pipeline including IndentFilter). This is the passthrough stub. The script is designed so later phases only change the case branch, not the interface.

Create tests/harness/rollback.sh (chmod +x).

Purpose: Confirm the F# reference pipeline is intact. Run this after any pipeline substitution experiment to verify the 368/442 fslit baseline is recoverable. This script operates on the LangThree repo, NOT on FunLexYacc.

Interface:
  rollback.sh [--langthree-dir <path>]
  Default LangThree dir: /Users/ohama/vibe-coding/LangThree

What it does:
1. Verifies the LangThree directory exists
2. Runs git checkout on Lexer.fs and Parser.fs (restores any generated substitutions)
3. Rebuilds LangThree in Release mode
4. Runs dotnet test and reports the F# unit test count (target: 199/199)
5. Runs fslit on the flt directory and reports results (target: 368/442 baseline)
6. Exits 0 if the build succeeds; exits 1 if build fails

IMPORTANT: This script does NOT modify anything in FunLexYacc. It only operates on the LangThree repository.

Full script:
```bash
#!/usr/bin/env bash
# rollback.sh — restore the F# LangThree pipeline and verify the known baseline
#
# Usage:
#   rollback.sh [--langthree-dir <path>]
#
# Default LangThree dir: /Users/ohama/vibe-coding/LangThree
#
# This script restores Lexer.fs and Parser.fs from git, rebuilds LangThree,
# and verifies the baseline test counts (199/199 unit, ~368/442 fslit).
# It does NOT modify the FunLexYacc repository.

set -euo pipefail

LANGTHREE_DIR="${LANGTHREE_DIR:-/Users/ohama/vibe-coding/LangThree}"

# Allow --langthree-dir override
while [[ $# -gt 0 ]]; do
    case "$1" in
        --langthree-dir)
            LANGTHREE_DIR="$2"
            shift 2
            ;;
        *)
            echo "ERROR: Unknown argument '$1'"
            exit 1
            ;;
    esac
done

if [ ! -d "$LANGTHREE_DIR" ]; then
    echo "ERROR: LangThree directory not found: $LANGTHREE_DIR"
    echo "Set LANGTHREE_DIR or pass --langthree-dir <path>"
    exit 1
fi

echo "=== rollback.sh: Restoring F# pipeline in $LANGTHREE_DIR ==="

# Step 1: git checkout to restore Lexer.fs and Parser.fs
echo ""
echo "--- Step 1: Restoring Lexer.fs and Parser.fs from git ---"
(cd "$LANGTHREE_DIR" && \
    git checkout src/LangThree/Lexer.fs src/LangThree/Parser.fs 2>/dev/null && \
    echo "Restored Lexer.fs and Parser.fs") || \
    echo "(Note: git checkout failed — files may not be modified, continuing)"

# Step 2: rebuild in Release mode
echo ""
echo "--- Step 2: Building LangThree (Release) ---"
dotnet build "$LANGTHREE_DIR/src/LangThree/" -c Release --nologo -q
echo "Build succeeded."

# Step 3: F# unit tests
echo ""
echo "--- Step 3: F# unit tests ---"
dotnet test "$LANGTHREE_DIR/tests/LangThree.Tests/" --no-build --nologo 2>&1 | tail -5

# Step 4: fslit integration tests
echo ""
echo "--- Step 4: fslit integration tests (expected baseline: ~368/442) ---"
FSLIT_BIN="${FSLIT_BIN:-fslit}"
if command -v "$FSLIT_BIN" >/dev/null 2>&1; then
    "$FSLIT_BIN" "$LANGTHREE_DIR/tests/flt/" 2>&1 | tail -5
else
    echo "WARNING: fslit not found in PATH. Skipping fslit step."
    echo "Install fslit or set FSLIT_BIN to the full path."
fi

echo ""
echo "=== rollback.sh complete ==="
```
